ASSESSMENT
This assessment is designed to evaluate engineering judgment, system design thinking, software quality, operational awareness, security considerations, and the ability to make thoughtful trade-offs in a real-world scenario.

You are encouraged to use AI-assisted development tools during this assessment. However, the evaluation will focus heavily on your technical decisions, architecture, operational thinking, handling of edge cases, and your ability to explain trade-offs and compromises.

The objective is not to build a perfect production system, but to demonstrate how you think as a senior engineer when designing and implementing a business-critical workflow.

SCENARIO
Build a small public seat reservation platform using any TypeScript stack with the following requirements.
The application should display 3 available seats that can be reserved by authenticated users.
Users should be able to:
Login with session expiry at 90 days
Select a seat
Proceed to payment
Successfully reserve the seat upon payment completion
Candidates should demonstrate practical engineering approaches that reflect common real-world scenarios, including thoughtful handling of trade-offs, operational concerns, failure cases, and system reliability.
You are free to choose your preferred:
Framework
Database
Infrastructure approach
Authentication provider
Mock payment provider
 Deployment method

SUBMISSION
Please submit your assessment in a zip file containing:
Source code
README documentation
Setup instructions
Environment configuration guidance
Any supporting scripts or documentation required to run the project locally

Typically, this system is expected to be completed within approximately 2 hours of focused implementation time. The assessment is designed to evaluate engineering judgment and technical decision-making rather than feature completeness or production polish.
